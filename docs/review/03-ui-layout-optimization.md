## 待优化

### 1. Layout 导航栏 — 小屏幕溢出

➡️ 跳过（桌面端窗口固定 ≥800px，无需处理）

---

### 2. AppCard 右侧按钮区域 — 对齐不整齐 ✅ 已修复

grid 布局固定列宽，右侧按钮始终在右上角。

---

### 3. CategoryDropdown 选项高亮 — 不够跳色 ✅ 已修复

选中项增加 ✓ 标记 + 触发按钮增加分类色边框微光。

---

### 4. History 双列布局 — 窄屏崩溃

➡️ 跳过（桌面端应用，不涉及窄屏）

---

### 5. Inventory 分类折叠 — 动画生硬 ✅ 已修复

改用 `max-height` + `opacity` transition 替代 `display: block/none` 切换。

---

### 6. 响应式布局适配 ✅ 已修复

| 适配项 | 方式 |
|--------|------|
| 导航栏 | 窄窗口时图标按钮（隐藏文字标签），横向可滚动 |
| Dashboard 统计卡片 | 窄窗口时 2 列显示（`@media max-width: 700px`） |
| History 双列布局 | 窄窗口时堆叠为单列（`@media max-width: 900px`） |
| 主内容区 padding | 窄窗口时从 32px 缩减为 16px |

---

### 7. 行内 style 泛滥

➡️ 已评估：全部行内 style 提取为 CSS 类工作量巨大，收益有限。当前行内 style 配合 CSS 变量体系已能满足主题需求，暂不处理。

---

### 8. 页面切换无过渡 ✅ 已修复

`key={page}` 触发 React 重建 + `@keyframes pageIn` fade + translateY 动画，每次切换页面时有平滑淡入效果。

---

### 9. Downloads 过滤按钮 — 选中态增强 ✅ 已修复

选中时增加 `scale(1.02)` 放大、`box-shadow` 阴影、底部指示条，与非选中态区分更明显。
